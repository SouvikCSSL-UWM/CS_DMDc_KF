## Overview

This repository implements a full sparse sensing and reduced-order state estimation pipeline, particularly suited to **high-dimensional physics-driven systems** where full-state measurement is impractical (e.g., temperature fields, fluid dynamics, structural monitoring).

The workflow proceeds in five sequential steps:

1. Reconstruct the full state from sparse spatial samples using Compressed Sensing
2. Learn a reduced-order dynamical model with external inputs via DMDc
3. Estimate initial process-noise covariance using EM
4. Perform online state estimation with a reduced-order Kalman filter
5. Adapt covariance online using DCC-GARCH

---

## Pipeline Summary

### Step 1 — Compressed Sensing Reconstruction (`CompressedSensing.py`)

Sparse spatial measurements of the full state are used to reconstruct the complete field in a transform domain.

- **Transform basis:** 2D Discrete Cosine Transform (DCT) / inverse DCT
- **Sampling strategy:** Random sparse spatial sampling
- **Optimization:** OWL-QN (via `pylbfgs`)

**Purpose:** Generate full-state approximations from undersampled measurements.

---

### Step 2 — Reduced-Order Dynamic Modeling with DMDc

The reconstructed state history is used to identify a reduced-order dynamical model incorporating control inputs using Dynamic Mode Decomposition with control (DMDc).

**Purpose:** Learn temporal system evolution in a lower-dimensional space while accounting for external forcing or actuation.

---

### Step 3 — Initial Covariance Estimation with EM

An Expectation-Maximization (EM) routine estimates the initial process-noise covariance for the reduced-order Kalman filter.

**Purpose:** Improve filter initialization in data-driven settings where process uncertainty is not known a priori.

---

### Step 4 — Online State Estimation with Kalman Filtering

A reduced-order Kalman filter performs online prediction and correction using sparse measurements, recovering the full field from latent reduced states.

**Purpose:** Real-time estimation of latent reduced state and full-field recovery.

---

### Step 5 — Time-Varying Covariance Update with DCC-GARCH

The process covariance is adaptively updated using Dynamic Conditional Correlation GARCH (DCC-GARCH) based on model residuals.

**Purpose:** Capture time-varying uncertainty and improve robustness when system dynamics or noise statistics change over time.

```

├── CompressedSensing.py           # Step 1: Sparse reconstruction via 2D DCT + OWL-QN
```

## Data Files

The pipeline expects the following CSV files in the working directory:

| File | Description | Typical Shape |
|------|-------------|---------------|
| `Temporature_history.csv` | Full-order ground truth state history (e.g., temperature field over a 2D grid) | `(n_states) × (n_timesteps)` |
| `Heat_input.csv` | Control / input signal history used by DMDc | `(n_inputs) × (n_timesteps)` |
| `Reconstructed_State.csv` | Full-state reconstruction generated by `CompressedSensing.py` | `(n_states) × (n_timesteps)` |

## Installation

Install all required dependencies with:

```bash
pip install numpy pandas matplotlib scipy pydmd arch opencv-python pylbfgs
```

| Package | Role |
|---------|------|
| `numpy` | Numerical computation |
| `pandas` | Data loading and manipulation |
| `matplotlib` | Visualization |
| `scipy` | Linear algebra and signal processing |
| `pydmd` | Dynamic Mode Decomposition |
| `arch` | DCC-GARCH covariance modeling |
| `opencv-python` | Image/field processing utilities |
| `pylbfgs` | OWL-QN optimization for compressed sensing |

The following standard library modules are also used and require no installation: `csv`, `warnings`, `collections`.

## How to Run

### Step 1: Run compressed sensing reconstruction

```bash
python CompressedSensing.py
```

This generates `Reconstructed_State.csv` and produces visualizations of the ground truth, reconstructed, and noisy fields.

### Step 2: Run the main estimation notebook

```bash
jupyter notebook CSDMD_KF_EM_DCCGARCH.ipynb
```

Execute cells in sequence to:

1. Load the data
2. Train the DMDc model
3. Estimate initial covariance via EM
4. Perform Kalman filtering in reduced space
5. Update covariance online with DCC-GARCH
6. Visualize and evaluate results

> **Note on file paths:** Some paths inside the scripts may need to be adjusted for your local machine and folder structure. Update any absolute paths before execution and verify package versions if compatibility issues arise.
